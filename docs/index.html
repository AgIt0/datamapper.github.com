Documentation Style

DataMapper uses the YARD standard for RDoc documentation. Here’s a brief sample from the internals of DM-Core:

 1 ##
 2 # Setups up a connection to a data-store
 3 #
 4 # @param [Symbol] name
 5 #   a name for the context, defaults to :default
 6 # @param [Hash(Symbol => String), Addressable::URI, String] uri_or_options
 7 #   connection information
 8 #
 9 # @return Repository
10 #   the resulting setup repository
11 #
12 # @raise ArgumentError "+name+ must be a Symbol, but was..."
13 #   indicates that an invalid argument was passed for name[Symbol]
14 # @raise [ArgumentError] "+uri_or_options+ must be a Hash, URI or String, but was..."
15 #   indicates that connection information could not be gleaned from
16 #   the given uri_or_options[Hash, Addressable::URI, String]
17 #
18 # @api public
19 def self.setup(name, uri_or_options)
20   raise ArgumentError, "+name+ must be a Symbol, but was #{name.class}", caller unless Symbol === name
21   ...

For more information about the YARD documentation style, see lsegal’s YARD repo on github.com.
